# Дилемма заключённого с памятью: как компаундинг общего счёта меняет баланс кооперации и предательства

*Исследование проведено на Python с воспроизводимым кодом. Seed: 42.*

---

## 1. Введение

В 1981 году Роберт Аксельрод провёл компьютерный турнир: программисты со всего мира прислали стратегии для повторяющейся дилеммы заключённого, и победил Анатоль Рапопорт с четырёхстрочной стратегией «Ты — мне, я — тебе» (Tit-for-Tat). Этот результат стал каноническим: при достаточно длинном горизонте взаимное сотрудничество выгоднее взаимной дефекции, и простые взаимные стратегии побеждают жадных.

Дилемма заключённого интересна именно тем, что создаёт противоречие между индивидуальной рациональностью и коллективным оптимумом. В единичном раунде дефектировать — строго доминирующая стратегия. В повторяющейся игре это перестаёт быть верным: теорема фолка показывает, что при достаточном «весе будущего» кооперативные равновесия могут поддерживаться. Стандартный способ моделировать «вес будущего» — дисконтирующий множитель δ < 1, который уменьшает ценность отдалённых раундов.

Мы предлагаем противоположную конструкцию: **мультипликатор, который растёт с историей**, а не убывает. Это соответствует ситуациям, где совместный успех создаёт ресурсы, усиливающие будущие взаимодействия: накопленный капитал генерирует доход, ресурсная база, выигранная кооперацией, повышает приспособленность потомства, или история взаимодействий увеличивает «вес» каждого следующего решения в долгосрочных отношениях. В таких системах **прошлое сотрудничество создаёт мультипликатор для будущих ставок**.

Классическая повторяющаяся PD устроена так: базовые выигрыши (CC=3, CD=0, DC=5, DD=1) суммируются раунд за раундом, и итог — просто накопленная сумма. Каждый раунд независим в том смысле, что прошлые выигрыши не влияют на *ценность* будущих очков. В нашей модификации это условие снимается: совокупный счёт S_(t) определяет, сколько «стоит» раунд t+1.

Компаундинг получается рекурсивным — S_(t+1) зависит от уже умноженных выигрышей, которые в свою очередь зависели от S_(t-1) и так далее. Это создаёт нелинейную обратную связь: партнёрства, которые генерируют большие потоки, разгоняют множитель быстрее и тем самым делают будущие раунды ещё ценнее. Ключевое наблюдение: взаимная кооперация (CC = 6 суммарных очков за раунд) разгоняет S быстрее, чем асимметричный исход DC (= 5) или взаимная дефекция DD (= 2).

Мы проверяли две гипотезы:

1. Компаундинг через M(S) делает кооперацию рационально выгодной даже против стратегий, не наказывающих дефекцию, начиная с некоторого горизонта T*.
2. Жёсткие непрощающие стратегии (Grim/Friedman) проигрывают прощающим значительно сильнее, чем в классической PD, — из-за потерянного будущего множителя после первой случайной ошибки.

## 2. Механика модификации

Базовая матрица выигрышей стандартная: CC = (3, 3), CD = (0, 5), DC = (5, 0), DD = (1, 1).

В каждом раунде t выигрыш игрока умножается на **M(S_t)**, где S_t — накопленный общий счёт обеих сторон (сумма уже умноженных выигрышей всех предыдущих раундов):

```
r_i(t) = payoff_i(a_t, b_t)          # сырой выигрыш из матрицы
p_i(t) = r_i(t) · M(S_t)             # умноженный выигрыш
S_{t+1} = S_t + p_A(t) + p_B(t)      # обновление накопленного счёта
```

Начальное состояние: S_0 = 0, поэтому M(0) = 1 при всех реализованных функциях — первый раунд всегда равен классической PD.

Ключевое наблюдение: сумма выигрышей по исходам составляет CC = 6, CD = DC = 5, DD = 2. Взаимная кооперация генерирует наибольший совокупный поток (6/раунд против 5 при асимметрии и 2 при взаимной дефекции). Это означает, что CC-партнёрство **разгоняет множитель быстрее**, чем любой другой исход.

Мы протестировали четыре формы M(S):

| Функция | Формула | Характер роста |
|---------|---------|----------------|
| Constant(1) | M = 1 | Классическая PD (контроль) |
| Linear(k) | M = 1 + S/k | Линейный, неограниченный |
| Power(k, α) | M = (1 + S/k)^α | Сублинейный при α < 1 |
| Log(k) | M = 1 + ln(1 + S/k) | Медленный, всегда растущий |
| Capped(M_inner, cap) | M = min(inner(S), cap) | Ограниченный сверху |

## 3. Аналитическое предсказание

Рассмотрим непрерывное приближение для пары AllC vs AllC (CC) и AllD vs AllC (DC, с точки зрения игрока A).

**Случай CC.** Каждый раунд S растёт на 6·M(S), а игрок A получает 3·M(S):

```
dS/dt = 6·(1 + S/k)    →    S_CC(t) = k·(e^{6t/k} − 1)
```

Кумулятивный выигрыш игрока A:

```
Cum_CC(t) = ∫₀ᵗ 3·e^{6τ/k} dτ = (k/2)·(e^{6t/k} − 1)
```

**Случай DC.** AllD получает 5·M(S), AllC получает 0; S растёт на 5·M(S):

```
dS/dt = 5·(1 + S/k)    →    S_DC(t) = k·(e^{5t/k} − 1)

Cum_DC(t) = ∫₀ᵗ 5·e^{5τ/k} dτ = k·(e^{5t/k} − 1)
```

**Условие кроссовера** Cum_CC(t*) = Cum_DC(t*):

```
(k/2)·(e^{6t/k} − 1) = k·(e^{5t/k} − 1)
```

Подстановка u = e^{t/k} даёт алгебраическое уравнение:

```
u^6 − 2u^5 + 1 = 0
```

Нетривиальный корень (u > 1) численно: **u ≈ 1.96**, откуда:

```
t* = k · ln(u) ≈ k · ln(1.96) ≈ 0.673 · k
```

Для k = 100: **t* ≈ 67 раундов** по непрерывному приближению.

Численный эксперимент (E4) даёт кроссовер на раунде **71** — разница в 6% объясняется дискретностью симуляции. Для других функций M(S) кроссовер CC vs DC в пределах 500 раундов отсутствует: Power(100, 0.5) и Log(100) растут слишком медленно, чтобы компаундинг CC успел обогнать преимущество DC в 5 очков против 3.

## 4. Методология

**Стек:** Python 3.14, NumPy, Pandas, Matplotlib, Seaborn, tqdm, pytest.

**Стратегии (10 штук):** AllC, AllD, Random(p=0.5), TitForTat, SuspiciousTfT, TitForTwoTats, GrimTrigger, GenerousTfT(forgiveness=0.1), ContriteTfT, Pavlov.

**Шум:** два режима — `implementation` (действие инвертируется, оба видят инвертированное) и `perception` (игрок видит чужое действие с ошибкой). Эксперименты E2 используют `implementation`.

**Метрики «победы»:** средний умноженный выигрыш на матч; ранг в round-robin; доля кооперации; итоговая доля в репликаторной динамике.

| Эксперимент | Параметры |
|-------------|-----------|
| E1 базовый турнир | Round-robin, T=200, noise=0, 100 повторений, 5 функций M(S) |
| E2 шум | Linear(100), T=200, noise ∈ {0, 0.01, 0.05, 0.1, 0.2}, 100 повторений |
| E3 горизонт | Linear(100), noise=0, T ∈ {10, 50, 100, 200, 500, 1000}, 100 повторений |

Все эксперименты воспроизводимы через `--seed 42`.

## 5. Результаты

### Эксперимент 1: базовый турнир

![E1: турнир](../pd_multiplier/output/figures/exp1_tournament.png)

**Constant(1) — контроль.** Ранжирование точно воспроизводит классический Аксельрод: GenerousTfT первый (537 очков), за ним TFT2 (535), TfT и ContriteTfT делят третье место (532). AllD последний (366). Разрыв GenerousTfT — AllD составляет 46%.

**Linear(100) — компаундинг.** Расстановка меняется кардинально. AllC поднимается с 7-го места на **1-е** (4 679 071 против 506 в Constant), AllD остаётся последним, но теперь отстаёт от лидера в **25 раз** (189 K против 4.7 M). Grim опускается с 5-го на 7-е место: его недостаточно высокая доля кооперации (70% против 87% у TFT2) обходится дороже при компаундинге.

Интересная деталь: доля кооперации у всех стратегий **не меняется** от множителя к множителю (TFT2 всегда 87%, AllD всегда 0%). Компаундинг не перепрограммирует стратегии — он только усиливает разницу в выигрышах. Именно поэтому разрыв между кооперативными и дефектными стратегиями растёт экспоненциально при нетривиальных M(S).

### Эксперимент 2: чувствительность к шуму

![E2: шум](../pd_multiplier/output/figures/exp2_noise.png)

Самый показательный результат всего исследования. При нулевом шуме Grim набирает 4 036 K — чуть хуже GenerousTfT (4 594 K). При шуме всего **1%** (noise = 0.01) Grim падает до 644 K — **снижение на 84%**. GenerousTfT при том же шуме теряет 30%, TfT — 50%.

При noise = 0.20 Grim сохраняет лишь 145 K (3.6% от своего бесшумного результата), тогда как GenerousTfT держит 658 K (14.3%). Разрыв между Grim и GenerousTfT, который при нуле составлял 14%, вырастает до 353% при максимальном шуме — GenerousTfT в 4.5 раза обходит Grim.

Механизм понятен: одна случайная дефекция навсегда включает триггер Grim. В классической PD это обходится потерей потока выигрышей от кооперации. В модели с компаундингом цена выше: теряется не просто поток, но и **рост множителя** — будущие раунды с высоким M(S) уже никогда не состоятся. Grim «замораживает» S на более низком уровне, чем прощающие стратегии.

Показательно сравнение AllC и Grim при шуме. AllC при noise = 0.01 теряет 23% (3 608 K против 4 679 K при нуле) — куда меньше Grim. Стратегия без памяти не может «обидеться» навсегда: каждый раунд она кооперирует независимо от прошлого. В мире с компаундингом это неожиданно выгодно: AllC не разрушает нарастающий множитель собственными ответными дефекциями. Разумеется, AllC при этом становится лёгкой добычей для AllD, но в разнородной популяции с преобладанием кооперативных агентов это не фатально.

### Эксперимент 3: чувствительность к горизонту

![E3: горизонт](../pd_multiplier/output/figures/exp3_horizon.png)

Кооперативные стратегии (AllC, TfT, GenerousTfT, Pavlov, TFT2, ContriteTfT) обгоняют дефектные (AllD, SuspiciousTfT) уже при **T = 10** раундов: средний выигрыш 33.3 против 25.3. Это неожиданный результат — в классической PD при T = 10 с фиксированным горизонтом дефекция рациональна из-за обратной индукции.

С ростом T разрыв увеличивается суперлинейно: при T = 1000 кооперативные стратегии опережают дефектные в **5 раз** (7.57 × 10²⁶ против 1.47 × 10²⁶). Экспоненциальный характер роста — прямое следствие рекурсивного компаундинга.

Почему T = 10 уже даёт преимущество кооперации? При Linear(100) в первые 10 раундов CC-партнёрство успевает разогнать S до ≈ 60 единиц, после чего M(S) ≈ 1.6. При этом кооперативные стратегии в паре друг с другом зарабатывают 3·1.6 = 4.8 за раунд, тогда как AllD против SuspiciousTfT большую часть матча играет DD (оба дефектируют), получая лишь 1·M. Суммарный выигрыш кооперативных стратегий усредняется по всем соперникам, и небольшой, но ненулевой кооперативный бонус перевешивает. Это контрастирует с классической PD, где разница фиксирована и обратная индукция убивает кооперацию на коротких горизонтах.

### Эксперимент 4: AllC vs AllC vs AllD (дуэль исходов)

![E4: дуэль](../pd_multiplier/output/figures/exp4_duel.png)

В голову-в-голову с AllD стратегия AllC всегда проигрывает: получая 0 очков каждый раунд (CD-выигрыш), она не может накапливать ничего независимо от M(S). Это честный результат, и мы его не скрываем.

Ключевое сравнение — **CC против DC**: кумулятивный выигрыш игрока A в паре AllC + AllC против его выигрыша в паре AllD + AllC.

- **Constant(1):** CC = 1 500, DC = 2 500. Дефектор всегда выигрывает. Кроссовера нет.
- **Linear(100):** кроссовер на раунде **71**. К раунду 500 CC = 2.25 × 10¹⁴, DC = 3.93 × 10¹². Взаимная кооперация опережает в **57 раз**.
- **Power(100, 0.5) и Log(100):** кроссовера нет в пределах 500 раундов, соотношения CC/DC = 0.70 и 0.63.

Вывод: численный T* = 71 хорошо согласуется с аналитическим предсказанием 67. Только линейный рост M(S) достаточно агрессивен, чтобы преимущество взаимной кооперации перевесило начальную фору дефектора (5 vs 3 в нулевой момент).

### Эксперимент 5: эволюционная динамика

![E5: эволюция](../pd_multiplier/output/figures/exp5_evolution.png)

Во всех четырёх условиях репликаторная динамика вытесняет AllD и SuspiciousTfT практически до нуля. При Constant(1) итоговая суммарная доля кооперативных стратегий — **87.5%**, AllD = 1.7 × 10⁻⁴³. При Linear(100) та же картина: 87.5% кооперация, AllD = 1.9 × 10⁻²⁵⁹ — значение столь мало, что в любой реальной конечной популяции AllD просто вымерла бы.

Интересно, что переход к нетривиальным множителям не меняет **качественную** структуру равновесия — кооперация доминирует везде. Количественно меняется порядок внутри кооперативного кластера: при Constant(1) лидирует GenerousTfT (16.2%), при Linear(100) — AllC (16.4%), что согласуется с E1.

Заметная деталь: Grim участвует в финальном кооперативном кластере наравне с другими (~12–13% доля), несмотря на слабость в шумной среде. Это объясняется тем, что E5 запускался без шума. В детерминированном мире Grim вполне эффективен; его уязвимость проявляется только при noise > 0. Это хорошо иллюстрирует, почему экологическая валидность эволюционных моделей зависит от предположений об окружении: одно и то же правило может быть «evolutionarily stable» в одних условиях и вымирать в других.

## 6. Обсуждение

### Судьба Grim

В чистой среде (E1, noise = 0) Grim работает достаточно хорошо: 7-е место при Linear, 5-е при Constant. Это объяснимо — без ошибок Grim кооперирует со всеми кооперирующими стратегиями и наказывает AllD. Но в шумной среде (E2) он коллапсирует катастрофически: при noise = 1% выигрыш падает на 84%, тогда как GenerousTfT теряет лишь 30%. При компаундинге цена каждой непрощённой ошибки — не просто потерянные раунды, а замороженный множитель. Grim хорош в стабильных детерминированных мирах; в реальных системах с любым уровнем шума он нежизнеспособен.

### Подъём AllC

AllC выигрывает в E1 при Linear(100) и занимает первое место в репликаторной эволюции — не потому что стала умнее, а потому что её партнёры по CC-матчам разгоняют множитель быстрее всех. Это важное наблюдение: компаундинг не делает наивную доброту рациональной в изоляции (в прямом противостоянии AllC/AllD AllC всегда получает ноль), но делает её рациональной в **среде, где большинство партнёров кооперируют**. Это близко к концепции assortative interaction в эволюционной биологии.

### TfT всё ещё силён?

Да, но иначе. В классическом турнире TfT и его варианты доминируют за счёт «наказательной симметрии» — дефектор немедленно получает отпор. При компаундинге добавляется второй канал: взаимная кооперация создаёт нарастающую ренту, которую дефекция обрывает. TFT2 и GenerousTfT держат высокую долю кооперации (87–86%) и выигрывают от этого рекурсивно. В итоге ранжирование верхней части таблицы сохраняется, но абсолютные разрывы многократно возрастают.

### Какая форма M(S) лучше?

Linear(100) даёт самые «кооперативные» равновесия в том смысле, что только при нём CC превышает DC в прямом противостоянии (кроссовер на раунде 71). Power(α = 0.5) и Log медленнее раскручивают множитель, поэтому преимущество от кооперации менее выражено — CC/DC к раунду 500 составляет 0.70 и 0.63 соответственно. Capped(Linear, 50) — промежуточный случай: после достижения планки компаундинг перестаёт работать и система вырождается в обычную PD с постоянным коэффициентом 50. Тем не менее Capped уже при cap = 50 заметно увеличивает абсолютные выигрыши и давление в пользу кооперации по сравнению с Constant(1).

Важный вывод: форма M(S) — это не технический параметр, а моделируемое предположение о природе среды. В экосистемах с линейным ростом ресурсов кооперация выгодна уже при T > 67. В средах с убывающей отдачей (Power, Log) порог либо выше, либо недостижим. Это подсказывает, что институциональный дизайн, управляющий «скоростью компаундинга» совместных ресурсов, может качественно менять равновесие между кооперативными и дефектными агентами в соответствующих социальных системах.

### Павлов: незамеченный победитель

Pavlov (Win-Stay Lose-Shift) держится в верхней части рейтинга во всех условиях: при Constant(1) он 6-й, при Linear(100) — 6-й. Стратегия не входит в топ-3, но обращает на себя внимание стабильностью. Pavlov не прощает так агрессивно, как GenerousTfT, и не запоминает дефекцию так жёстко, как Grim, — он просто повторяет успешное поведение и меняет провальное. В шумной среде эта адаптивность позволяет ему держаться лучше, чем Grim, хотя и хуже, чем GenerousTfT.

## 7. Ограничения

**Конечный горизонт** фиксирован заранее. В реальных системах с неизвестным горизонтом может срабатывать обратная индукция; в нашей модели игроки горизонт знают, но не используют его стратегически (за исключением потенциального end-game поведения, которое простые стратегии не реализуют).

**Отсутствие коэволюции стратегий.** Репликатор меняет *доли* стратегий, но не их *параметры*. Grim с forgiveness=0 соревнуется с Grim-like стратегиями, но не может «научиться» прощать. Добавление мутаций изменило бы долгосрочное равновесие.

**Фиксированная M(S) на протяжении всей игры.** В реальности участники могут иметь разные представления об эффективной ставке дисконтирования / компаундирования. Эндогенный выбор M(S) — открытый вопрос.

**Только два игрока.** Групповые дилеммы (n > 2) принципиально отличаются: при большом числе игроков условие взаимности сложнее выполнить, а компаундинг может создать новые проблемы free-rider.

## 8. Дальнейшая работа

**Мета-стратегии с выбором M(S).** Если игроки могут предлагать или отвергать правила компаундинга до начала игры, это создаёт игру второго уровня с потенциально богатой структурой равновесий.

**Многоагентные группы.** Расширение на n > 2 игроков с общим пулом, где совокупный вклад формирует М(S) для всего пула — прямая связь с моделями публичных благ.

**Адаптивные стратегии.** Эволюционные нейронные сети или Q-learning агенты, способные адаптировать поведение к текущему значению S и M(S), вероятно, найдут стратегии, недоступные нашему зоопарку.

**Несимметричные функции.** Разные M_A(S) и M_B(S) для двух игроков (например, разные ставки дисконтирования) — формализация асимметрии интересов в долгосрочных контрактах.

### Псевдокод движка

```
для каждого раунда t = 0..T-1:
    a = player_A.play(my_hist_A[:t], opp_hist_A[:t], t, T)
    b = player_B.play(my_hist_B[:t], opp_hist_B[:t], t, T)
    если noise > 0: применить шум к (a, b)
    r_A, r_B = payoff_matrix[a][b]
    m = M(S[t])
    p_A, p_B = r_A * m, r_B * m
    S[t+1] = S[t] + p_A + p_B
```
