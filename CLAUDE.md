This file provides guidance to Claude Code (claude.ai/code) when working with code in this repository.

## Что это за репозиторий

Репозиторий **только с контентом** для блога `kami.letar.best`. Здесь нет ничего, кроме постов (`.mdoc` и `.yaml` в `content/posts/`) и `LICENSE` (CC BY 4.0). Фронтенд на Next.js + Keystatic + Chakra UI живёт в отдельной "lena monorepo" — его **нет** в этом репозитории.

Правки приходят двумя путями: ручные коммиты и Keystatic admin UI (он автоматически создаёт коммиты вида `Update content/posts/<slug>`).

## Здесь нет build / test / lint

Нет `package.json`, скриптов, тестов, линтера. Не ищи их. Единственная осмысленная проверка изменения контента — "рендерится ли это во фронтенд-репо"; если нужно проверить, это делается там, не здесь.

## Структура постов

Посты используют Markdoc (`.mdoc`) — формат Keystatic, не обычный MDX. Сосуществуют два варианта:

- **Один файл**: `content/posts/<slug>.mdoc` с YAML-фронтматтером сверху и телом ниже. Пример: [akczentuaczii-haraktera.mdoc](content/posts/akczentuaczii-haraktera.mdoc).
- **Папка**: `content/posts/<slug>/content.mdoc` (только тело) + `content/posts/<slug>/index.yaml` (только фронтматтер). Пример: [hello-world](content/posts/hello-world/). Использовать когда у поста есть соседний файл с английским переводом или нужно держать рядом ассеты.

### Поля фронтматтера

| Поле            | Тип            | Заметки                                     |
| --------------- | -------------- | ------------------------------------------- |
| `title`         | строка         | Русский заголовок, обязательно              |
| `titleEn`       | строка         | Английский заголовок (пока только в папках) |
| `description`   | строка         | Русское описание, обязательно               |
| `descriptionEn` | строка         | Английское описание (пока только в папках)  |
| `publishedAt`   | ISO-дата       | например `2026-01-01` или `'2025-12-08'`    |
| `tags`          | список строк   | например `- психология`                     |
| `featured`      | булево         | `true` закрепляет пост в featured-слоте     |

## Двуязычный контент

Русский — основной язык. Английские переводы лежат в соседнем файле: `content/posts/<slug>/contentEn.mdoc`. У обоих длинных постов сейчас пустые `contentEn.mdoc` — перевод ещё не сделан. У "одно-файлового" варианта нет места под английское тело; если посту нужен перевод, сначала мигрируй его в папочный вариант.
